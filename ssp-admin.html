SSP Admin Management
기능 기획 및 UX/UI 설계

SaaS 운영 환경에서 필요한 Tenant, Company, Data 관리 기능을 기획하고, Admin 사용자가 조직·테넌트·데이터 구조를 효율적으로 관리할 수 있도록 UX/UI를 설계한 프로젝트입니다.

Admin UX Designer
Scope Admin IA, UX/UI Design
User Flow, Dev QA

Company Management
Data Management

Background

SSP Admin은 테넌트, 회사, 데이터 구조를 관리해야 하는 SaaS 운영 플랫폼입니다. 기능별 요구사항은 존재했지만, 실제 사용자가 정보를 탐색하고 설정을 변경하는 화면 구조와 관리 흐름은 구체화가 필요한 상태였습니다.

Admin 사용자가 복잡한 조직·테넌트·데이터 정보를 쉽게 탐색하고 관리할 수 있도록, 기능별 IA와 화면 구조를 정리하는 과정이 필요했습니다.

Problem
  • Tenant, Company, Data 관리 기능의 IA가 명확히 정리되지 않음
  • Detail 화면에서 어떤 정보를 어떤 탭으로 구분할지 기준이 필요함
  • Data Management는 데이터 구조가 복잡해 사용자가 이해하기 쉬운 메뉴 설계가 필요함
  • 개발 구현 결과와 디자인 산출물 간 검수 프로세스가 필요함
Goal

Admin 사용자가 테넌트, 회사, 데이터 정보를 쉽게 탐색하고 관리할 수 있도록 기능 구조를 정리하고, 개발 가능한 수준의 화면 설계와 유저 플로우를 제공하는 것을 목표로 했습니다.

01

Admin IA 구조화

Tenant, Company, Data 관리 기능을 사용 목적과 정보 구조에 맞게 분류하고 메뉴 체계를 정리합니다.

02

관리 흐름 설계

List, Detail, Tab 구조를 기반으로 Admin 사용자가 필요한 정보를 탐색하고 설정을 변경할 수 있는 흐름을 설계합니다.

03

데이터 관리 명확화

복잡한 데이터 관리 개념을 Data Ingestion 유저 플로우와 화면 단위로 구체화합니다.

04

개발 연계

개발팀 요구사항을 확인하고 구현 화면을 검수하여 디자인 산출물이 실제 제품 구현까지 연결되도록 합니다.

My Role

Product Designer이자 Admin UX Designer로서 SaaS Admin 관리 기능의 IA 설계, 화면 기획, UX/UI 디자인, 데이터 관리 유저 플로우 작성, 개발 리뷰 및 구현 화면 검수까지 담당했습니다.

  • SaaS Admin 관리 기능 IA 및 화면 구조 설계
  • Tenant Management List / Detail 화면 기획 및 디자인
  • Company Management 화면 개선 및 변경사항 반영
  • Data Management 메뉴 구조 및 Data Ingestion 유저 플로우 설계
  • 개발팀 요구사항 확인 및 기획 범위 조율
  • 개발 구현 화면 View / Edit 기능 검수
  • 로그인 화면 커스터마이징 및 Import Stores 기능 설계 대응
Process
01

Requirement Review

기존 요구사항과 기능 범위를 확인하고, Admin 관리 기능에서 필요한 화면 구조를 검토했습니다.

02

Management Feature Classification

Tenant, Company, Data 관리 기능을 구분하고 각 기능의 목적과 정보 구조를 정리했습니다.

03

List / Detail / Tab IA Design

Tenant Management를 중심으로 List, Detail, General, Store, Design 탭 구조를 설계했습니다.

04

SaaS Data Reference Research

SaaS 데이터 관리 솔루션을 리서치하며 Data Management 메뉴와 화면 흐름에 필요한 기준을 검토했습니다.

05

Data Ingestion User Flow

Data Ingestion 흐름을 중심으로 Admin 사용자가 데이터를 이해하고 관리할 수 있는 유저 플로우를 작성했습니다.

06

UX/UI Design Application

정리된 IA와 유저 플로우를 바탕으로 Tenant, Company, Data Management 화면 설계와 UI 디자인을 적용했습니다.

07

Development Review

개발팀 요구사항을 확인하고 기획 범위를 조율하며, 리뷰 과정에서 변경사항을 반영했습니다.

08

View / Edit QA

개발 완료된 Map View / Editor의 View/Edit 기능을 검수하고 구현 상태를 확인했습니다.

Key Work

SaaS Admin 환경에서 테넌트 정보를 효율적으로 관리할 수 있도록 Tenant Management 기능을 기획하고 화면을 설계했습니다. List와 Detail 구조를 중심으로 정보 탐색 흐름을 정리하고, 사업자 인증, Store Import, Login 커스터마이징 등 실제 운영에 필요한 관리 기능을 탭 구조로 구성했습니다.

  • Tenant List 화면 기획 및 디자인
  • Tenant Detail Page 세부 항목 기획
  • General 탭 내 사업자 인증 관련 화면 설계
  • Store 탭 내 Import Stores 기능 설계
  • Design 탭 내 Login 화면 커스터마이징 설계
  • 개발 리뷰 대응 및 변경사항 반영

복잡한 데이터 관리 구조를 Admin 사용자가 이해하기 쉽도록 Data Management 메뉴와 화면 흐름을 설계했습니다. SaaS 데이터 관리 솔루션을 리서치하고, 개발팀과 데이터 구조 방향성을 조율하여 Data Ingestion 중심의 유저 플로우와 IA를 구체화했습니다.

  • SaaS 데이터 관리 솔루션 레퍼런스 리서치
  • STI / SSP 데이터 관리 구조 초안 요청 및 요구사항 확인
  • Data Management 방향성 정리 회의 참여
  • 메뉴 기능 관련 개발 요구사항 확인
  • 변경된 IA 기반 메뉴 상세 설계
  • Data Ingestion 유저 플로우 작성
  • Data Management 화면 설계 및 디자인 적용

Company Management 화면의 기획 누락 및 변경사항을 점검하고, UI 컴포넌트 구조를 정리했습니다. 기존 화면의 관리 기능 흐름을 유지하면서 변경된 요구사항이 일관된 UI 패턴으로 반영되도록 개선했습니다.

  • 기획 누락 항목 확인
  • 변경사항 디자인 반영
  • 디자인 컴포넌트 변경 및 UI 정리
  • Company Management 화면 구조 개선

개발 완료된 Map View / Editor 기능의 View/Edit 동작을 검수하며 디자인 산출물과 구현 결과 간의 차이를 확인했습니다. 최종 개발안 기준으로 기능 완성도를 점검하고, QA 관점에서 개발 대응을 진행했습니다.

  • Seah Lab Map View / Editor 개발 대응
  • 최종 개발안 View 기능 검수
  • Edit 기능 검수
  • 개발 완료 상태 확인
Outcome

Tenant, Company, Data Management 기능의 화면 구조와 관리 흐름을 구체화하여 SaaS Admin의 운영 기능 기반을 정리했습니다. 또한 개발팀과의 검토 및 기능 검수를 통해 디자인 산출물이 실제 구현 단계까지 연결될 수 있도록 기여했습니다.

  • SaaS Admin 핵심 관리 기능의 화면 구조 구체화
  • Tenant / Company / Data 관리 기능의 IA와 UX 흐름 정리
  • 복잡한 데이터 관리 개념을 Admin 화면 단위로 구조화
  • 개발팀 요구사항과 기획 범위를 조율하여 구현 가능성 강화
  • View / Edit 개발 검수를 통해 디자인-개발 간 품질 차이 최소화
  • Admin 제품군 내 관리 기능의 UI 일관성 확보
